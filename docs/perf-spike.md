# TypeVoice ASR 性能验证计划

目标：用可复现数据验证当前 ASR provider 的转录耗时、取消响应与 FFmpeg 预处理耗时。

## 1. 验证范围

- FFmpeg 预处理耗时。
- Doubao 流式 ASR 或远程 HTTP ASR 的转录耗时。
- LLM 改写网络耗时单独统计展示。

## 2. 核心指标

- RTF = ASR 转录耗时 / 音频时长。
- 总耗时：预处理耗时 + provider 转录耗时。
- 取消响应：用户触发取消到状态完成更新的耗时。

## 3. 测试样本集

- A：10 秒，中文短句，含少量停顿。
- B：60 秒，中文段落，口语表达，含数字。
- C：5 分钟，中文连续表达，含停顿与自我修正。

本机样本文件约定与来源见：`docs/fixtures-sources.md`。

## 4. 预处理口径

- 使用内置 FFmpeg 统一输出为 WAV/PCM/16kHz/mono。
- 预处理参数固定后通过验证脚本复用。

## 5. 决策输出

- 当前 provider 的 RTF 与取消响应是否达标。
- 主要瓶颈点。
- 下一步优化建议。
